/App/

Основной каталог кода приложения. Содержимое данного каталога проецируется в память в пространство имен Application.

Каждый каталог объявляется как пустой объект {},

файлы с расширением html объявляются как текстовые переменные,

файлы с расширением js подключаются как исполняемый код через require().

Остальные файлы данного каталога игнорируются.

Например, класс из файла по пути App/Controller/Welcome.js станет объектом в памяти Application.Controller.Welcome

Каталог имеет следующую структуру:

Model/

View/

Controller/

Middleware/

Library/

Sheduler/

database.js

routes.js

mailer.js

Model/

Каталог классов моделей данных

View/

Каталог шаблонов html

Controller/

Каталог контроллеров запросов

Middleware/

Каталог промежуточных обработчиков

Library/

Каталог коллекций данных или программного кода, используемые в приложении

Sheduler/

Каталог планировщика задач. Содержит подкаталоги Periodic и StartUp.

Код, размещенный в StartUp, выполняется при старте сервера.

Код, размещенный в Periodic, выполняется по указанному расписанию

database.js

Файл конфигурации баз данных

routes.js

Файл конфигурации роутингов доменов

mailer.js

Файл конфигурации исходящей почты

/Public/

Корневой каталог статических файлов, доступных по прямым ссылкам из браузера. Скрипты js css и картинки должны размещаться здесь

/System/

Каталог системных модулей приложения

/config.ini

Конфигурационный файл приложения. Загружается в неймспейс Application.config и доступен из любого места приложения.

/main.js

Основной загрузочный файл приложения, выполняющий загрузку и запуск в работу.

/npmlibs.js

Модули npm, используемые в приложении. Все модули инициируются в неймспейс Application.lib и доступны глобально

/package.json

Конфигурационный файл node.js, содержащий перечень необходимых npm модулей

/systemd.service

Пример конфигурационного файла для установки приложения как службы systemd. В нем необходимо скорректировать пути, затем установить симлинк в систему и инициировать
